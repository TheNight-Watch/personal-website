This project is not a traditional app. It is a workflow skill for agentic iOS development.

## Starting point

I kept seeing the same problem: when agents work on iOS code, details like environment propagation, CodingKeys alignment, testing discipline, and visual review are easy to skip. So I tried to turn those lessons into a staged orchestration method with real constraints.

## Core content

- Study the design principles behind strong agent workflows
- Extract hard gates and checkpoints that fit iOS engineering
- Represent the whole system with markdown and state transitions instead of a heavy platform
- Make clarification, TDD, regression checks, and adversarial review default behavior

## Value

The goal is to create a reusable discipline layer, so an agent does not merely write code, but can complete an iOS delivery chain more reliably end to end.
