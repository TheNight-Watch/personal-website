Agent Manager is shaped by a very specific work pattern: I often run many Claude Code agents in parallel, and those sessions are scattered across different workspaces. Managing them quickly becomes messy.

## Goal

I want an interface that organizes agents by workspace and makes it easy to see their status, progress, and possible actions in one place.

## What matters

The goal is not to wrap a terminal in a prettier shell. The real question is which parts of agent work deserve a shared management layer, and which parts should stay native to the existing environment.

## Current work

Right now the project is mostly research: comparing existing solutions and deciding whether a new product is justified, or whether a lighter extension layer would be enough.
