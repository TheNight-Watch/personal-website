这个项目并不是一个传统应用，而是一套面向 iOS 开发的 Agent 工作流 Skill 设计。

## 起点

我观察到，当 Agent 参与 iOS 开发时，Environment 传播、CodingKeys 对齐、测试纪律和视觉审查这些细节非常容易被跳过。于是我尝试把这些经验抽象成一套有阶段约束的开发编排方法。

## 核心内容

- 研究高质量 Agent workflow 的设计原则
- 提炼适用于 iOS 工程的硬约束与阶段门槛
- 用 markdown 和状态机组织完整流程，而不是另造一套复杂平台
- 把需求澄清、TDD、回归验证和对抗式审查都前置成默认动作

## 项目价值

我希望它最终成为一个“可复制的开发纪律层”，让 Agent 不是只会写代码，而是能更可靠地完成整条 iOS 交付链路。
